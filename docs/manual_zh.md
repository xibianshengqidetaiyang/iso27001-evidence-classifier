# ISO27001 证据分类脚本使用手册（GitHub版）

## 1. 项目简介

本项目用于对 ISO/IEC 27001 审核过程中的证据材料进行**自动初筛与多对多分类**。  
脚本会同时分析**文件名**与**正文内容**，并结合审核手册中预先整理的规则，将一个证据文件匹配到一个或多个控制项；同一个控制项也可以对应多个证据文件。

它适合做的事情包括：

- 审核前的证据归档与初步分类
- 审核中对资料完整性的辅助检查
- 审核规则库的结构化沉淀
- 作为安全治理 / 安全合规 / GRC 方向的个人作品集项目

> 注意：当前版本是**规则驱动的证据分类工具**，不是最终的合规判定引擎。它适合做“初筛”和“归类”，不直接替代人工审核结论。

---

## 2. 核心功能

### 2.1 多对多分类

- 一个证据文件可以同时匹配多个 ISO 27001 控制项
- 一个控制项也可以聚合多个证据文件

### 2.2 同时看文件名与正文

即使文件名不标准，只要正文中包含与控制项相关的内容，也可以被分类到对应控制项下。

### 2.3 基于审核手册自动抽取规则

脚本会从结构化的审核手册中提取：

- 控制项编号
- 控制项标题
- 一票否决必填证据清单
- 核心关键词
- 辅助关键词

再据此建立分类规则。

### 2.4 输出命中理由与置信度

输出结果不仅包含“分到了哪里”，还会包含：

- 命中的关键词
- 命中的证据描述
- 片段摘要
- 分数
- 置信度

这有利于人工复核。

### 2.5 支持多种常见证据格式

当前支持：

- `.md`
- `.txt`
- `.docx`
- `.pdf`（需为可提取文本的 PDF）
- `.xlsx`
- `.csv`
- `.json`

## 3. 适用场景

适合的证据材料包括：

- 管理制度

- 台账

## 7. 使用前准备

### 7.1 准备证据目录

新建一个 `evidence` 文件夹，把需要分类的证据文件放进去。可以包含多层子目录，脚本会递归扫描。

### 7.2 确认两个审核手册文件存在

需要确保以下两份 Markdown 文件存在，且路径正确：

- `ISO27001_2022_信息安全管理体系总则审核手册_V1.0.md`
- `ISO27001_2022_附录A_A5-A8_控制项审核手册_V1.0.md`

### 7.3 审核手册格式要求

为保证规则抽取效果，手册最好保持结构化格式，例如包含：

- `### A.5.7 xxx`
- `### 4.1 xxx`
- `【一票否决必填证据清单】`
- `核心关键词`
- `辅助关键词`

如果手册结构变化过大，规则提取效果会下降。

## 8. 快速开始

### 8.1 推荐做法

在三个文件夹的**共同上级目录**中打开终端，然后执行脚本，并通过 `--handbooks` 显式传入两个手册路径。

### 8.2 PowerShell 命令

```powershell
python ".\iso27001_evidence_classifier_package\iso27001_evidence_classifier.py" `
  --evidence-dir ".\evidence" `
  --output-dir ".\iso27001_evidence_classifier_package\output" `
  --handbooks `
  ".\ISO27001_2022_审核手册与模板_交付包\ISO27001_2022_信息安全管理体系总则审核手册_V1.0.md" `
  ".\ISO27001_2022_附录A_A5-A8_审核手册与规则库_交付包\ISO27001_2022_附录A_A5-A8_控制项审核手册_V1.0.md" `
  --threshold 3.5
```

### 8.3 CMD 命令

```cmd
python ".\iso27001_evidence_classifier_package\iso27001_evidence_classifier.py" ^
  --evidence-dir ".\evidence" ^
  --output-dir ".\iso27001_evidence_classifier_package\output" ^
  --handbooks ^
  ".\ISO27001_2022_审核手册与模板_交付包\ISO27001_2022_信息安全管理体系总则审核手册_V1.0.md" ^
  ".\ISO27001_2022_附录A_A5-A8_审核手册与规则库_交付包\ISO27001_2022_附录A_A5-A8_控制项审核手册_V1.0.md" ^
  --threshold 3.5
```

## 9. 参数说明

### 9.1 `--evidence-dir`

待分类证据目录。

示例：

```bash
--evidence-dir ".\evidence"
```

### 9.2 `--output-dir`

输出目录。

示例：

```bash
--output-dir ".\iso27001_evidence_classifier_package\output"
```

### 9.3 `--handbooks`

用于传入一个或多个审核手册路径。你的场景下应传入两份 Markdown 手册。

示例：

```bash
--handbooks "路径1" "路径2"
```

### 9.4 `--threshold`

分类阈值，默认建议 `3.5`。

含义：

- 阈值较低：命中更多，但误报可能增加
- 阈值较高：更严格，但可能漏掉边缘匹配

建议：

- 命中文件太少：可尝试 `2.8 ~ 3.2`
- 命中结果太乱：可尝试 `4.0 ~ 5.0`

## 10. 输出结果说明

运行完成后，`output` 目录中通常会生成以下文件：

### 10.1 `classification_detail.csv`

最重要的明细结果，包含：

- 文件路径
- 控制项编号
- 控制项标题
- 分数
- 置信度
- 命中的关键词
- 命中的理由

适合人工复核。

### 10.2 `classification_summary.json`

按文件聚合的摘要结果，适合二次开发或程序读取。

### 10.3 `classification_by_control.csv`

按控制项聚合证据。适合审核人员查看某个控制项下命中了哪些文件。

### 10.4 `compiled_rules.csv`

从审核手册中自动提取出来的规则。可用于检查：

- 某控制项是否被正确提取
- 关键词是否被正确抽取
- 必填证据清单是否有效

### 10.5 `report.md`

简要总结报告，方便快速浏览本次分类结果。

## 11. 分类逻辑说明

脚本主要基于以下信号进行评分：

1. 文件名中的关键词命中
2. 正文中的核心关键词命中
3. 正文中的辅助关键词命中
4. 必填证据描述的相关内容命中
5. 文本片段的重叠程度

最终会给每个“文件 - 控制项”对计算一个分数。只要分数超过阈值，就认为该文件可以归到对应控制项下。

因此，该工具天然支持：

- **一份证据命中多个控制项**
- **一个控制项命中多份证据**

## 12. 首次使用建议

第一次使用时，不建议直接导入几百份文件。推荐做法：

1. 先选 10~20 份你熟悉的证据文件
2. 跑一次脚本
3. 查看 `classification_detail.csv`
4. 人工检查以下三点：
   - 文件名不规范，但正文相关时，是否能命中
   - 一个文件是否能命中多个控制项
   - 同一控制项是否能聚合多份文件

如果这三点都正常，说明脚本已经具备基础可用性。

## 13. 常见问题排查

### 13.1 `python` 命令不可用

原因：Python 未安装或未加入环境变量。

处理：重新安装 Python，并勾选“Add Python to PATH”。

### 13.2 路径中有中文或空格导致失败

处理：所有路径都用双引号包起来。

正确示例：

```bash
".\ISO27001_2022_审核手册与模板_交付包\ISO27001_2022_信息安全管理体系总则审核手册_V1.0.md"
```

### 13.3 手册文件名不一致

处理：确认命令中的文件名与实际文件名完全一致。

### 13.4 证据目录写成了单个文件路径

处理：`--evidence-dir` 必须指向文件夹，而不是单个文件。

### 13.5 PDF 无法识别内容

原因：PDF 可能是扫描件。

处理：

- 转成可搜索 PDF
- 或后续增加 OCR 流程

### 13.6 Excel 内容过杂导致误报

原因：Excel 中包含大量无关列、无意义数据或噪声。

处理：先做数据清洗，再交给分类器。
