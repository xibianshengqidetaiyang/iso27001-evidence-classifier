# Case Study

本文档用于展示本项目在脱敏样例证据上的处理流程，帮助读者快速理解：
输入是什么、规则如何命中、候选控制项如何聚合、AI 初步审核会输出什么、人工最终还需要复核什么。

---

## Case 1：员工手册类证据

### 输入
- 文件名：`1.员工手册.pdf`
- 证据类型：制度/管理类材料

### 规则命中思路
- 员工行为规范
- 安全职责要求
- 培训/宣导相关表述
- 人员管理制度关键词

### 候选控制项
- 人员安全相关控制项
- 安全意识与培训相关控制项
- 组织层面的管理要求相关控制项

### AI 初步审核输出关注点
- 是否判断为“基本相关”或“部分相关”
- 是否提示“缺少正式发布记录”
- 是否提示“缺少培训签到/宣导留痕”
- 是否提示“缺少版本信息或审批痕迹”

### 人工复核点
- 是否是正式生效版本
- 是否有发布日期、版本号、审批信息
- 是否能和培训记录或签收记录形成闭环
- 是否能证明制度已落地执行，而不只是存在文档

## Case 2：信息资产分类分级管理程序

### 输入
- 文件名：`77.3.036 信息资产分类分级管理程序.docx`
- 证据类型：制度/流程类材料

### 规则命中思路
- 信息资产
- 分类分级
- 资产识别
- 敏感程度/重要性分级

### 候选控制项
- 信息资产管理相关控制项
- 信息分类与标记相关控制项
- 资产责任划分相关控制项

### AI 初步审核输出关注点
- 是否能识别出该证据属于“资产管理制度”
- 是否提示“仅有制度，缺少实际资产台账”
- 是否提示“缺少执行记录或样例”
- 是否提示“缺少责任人和定期复核机制”

### 人工复核点
- 是否有配套资产清单/台账
- 是否有分类分级结果样例
- 是否能看到责任人、审批人、更新时间
- 是否能证明制度已被执行，而不是只有程序文件

## Case 3：日志检查评审记录

### 输入
- 文件名：`日志检查评审记录1-8月.pdf`
- 证据类型：记录/执行留痕类材料

### 规则命中思路
- 日志检查
- 审查/评审
- 监控记录
- 安全事件发现与跟踪

### 候选控制项
- 日志记录与监控相关控制项
- 安全事件识别相关控制项
- 运行审计/持续检查相关控制项

### AI 初步审核输出关注点
- 是否判断为“执行记录类证据”
- 是否提示“覆盖周期是否充分”
- 是否提示“缺少责任人/处置结果”
- 是否提示“缺少异常项闭环说明”

### 人工复核点
- 日志检查是否持续发生
- 是否有明确检查人、检查日期、检查结果
- 是否包含异常项和整改跟踪
- 是否能和制度要求相互印证

---

## 结论

通过以上样例可以看到，本项目的价值不在于直接替代人工审计判断，
而在于先把证据做初筛、候选控制项聚合、结构化初审和缺口提示，
帮助人工更快进入复核阶段。
